# 📊 배당 캘린더 웹앱 기획서

> KOSPI · KOSDAQ 상장 기업의 배당 입금일과 배당락일을 한눈에 확인할 수 있는 웹 서비스

---

## 1. 프로젝트 개요

### 1.1 배경 및 목적

국내 주식 투자자들이 배당 관련 일정(배당 입금일, 배당락일)을 파편화된 여러 사이트를 돌아다니며 확인해야 하는 불편함을 해소하기 위한 서비스입니다. KOSPI 및 KOSDAQ에 상장된 기업들의 배당 일정을 달력 형태로 시각화하여 제공하고, 종목별 배당 히스토리와 증감 추이까지 확인할 수 있는 웹앱을 개발합니다.

### 1.2 핵심 가치

- **한눈에 보는 배당 일정**: 달력 UI를 통해 월별 배당 입금일/배당락일을 직관적으로 확인
- **종목별 배당 히스토리**: 과거 배당금 내역과 증감률을 통해 투자 판단에 도움
- **시가총액 기반 우선 표시**: 주요 대형주 배당 일정을 우선 노출하여 정보 과부하 방지

### 1.3 기술 스택 (권장)

| 구분 | 기술 |
|------|------|
| 프론트엔드 | Next.js (App Router) + TypeScript |
| 스타일링 | Tailwind CSS + shadcn/ui |
| 상태관리 | Zustand 또는 React Query (TanStack Query) |
| 백엔드/API | Next.js API Routes (서버리스) |
| DB | Supabase (PostgreSQL) 또는 PlanetScale |
| 배포 | Vercel |
| 개발 도구 | Claude Code |

## 2. 데이터 소스

배당 정보를 수집하기 위해 아래 공식 API/데이터 소스를 활용합니다.

### 2.1 주요 데이터 소스

#### ① 공공데이터포털 - 금융위원회 주식배당정보 API (★ 핵심)

- **URL**: `https://www.data.go.kr/data/15043284/openapi.do`
- **엔드포인트**: `http://apis.data.go.kr/1160100/service/GetStocDiviInfoService/getDiviInfo`
- **제공 데이터**: 배당기준일자, **현금배당지급일자(= 배당 입금일)**, 주식교부일자, 주식일반배당금액, 차등배당금액, 배당률, 주식종류(보통주/우선주), 배당사유 코드 등
- **갱신 주기**: 일 1회 (영업일 기준 익일 오후 1시 이후)
- **인증**: API 키 발급 필요 (무료, 개발 계정 10,000건/일)
- **비고**: 법인등록번호 또는 주식발행회사명으로 조회 가능

#### ② 공공데이터포털 - 금융위원회 공시정보 API (배당공시)

- **URL**: `https://www.data.go.kr/data/15059649/openapi.do`
- **제공 데이터**: 배당공시정보 (배당 결정 공시, 주주총회 관련 정보)
- **용도**: 배당 결정 공시 시점을 추적하여 예정된 배당 일정 파악

#### ③ DART OpenAPI (금융감독원 전자공시시스템)

- **URL**: `https://opendart.fss.or.kr/`
- **제공 데이터**: 사업보고서 내 "배당에 관한 사항" (과거 배당 이력, 배당성향, 배당수익률)
- **용도**: 종목별 과거 배당금 히스토리 및 증감 추이 분석
- **인증**: API 키 발급 필요 (무료)

#### ④ KRX 한국거래소 (KIND 배당정보)

- **URL**: `https://kind.krx.co.kr/disclosureinfo/dividendinfo.do`
- **제공 데이터**: 2010년부터 최근 사업연도까지 상장법인 배당 관련 정보
- **용도**: 배당 데이터 보완 및 검증용 (스크래핑 필요)

#### ⑤ KRX Data Marketplace / KRX Open API

- **URL**: `https://data.krx.co.kr/` / `https://openapi.krx.co.kr/`
- **제공 데이터**: 시가총액, PER/PBR/배당수익률, 종목 기본정보
- **용도**: 시가총액 순위 기반 정렬, 종목 기본정보 매핑

### 2.2 배당락일 산출 방식

> ⚠️ **배당락일은 별도 API로 직접 제공되지 않는 경우가 많습니다.**

배당락일은 아래 규칙으로 산출합니다:

- **배당락일 = 배당기준일(Record Date)의 1영업일 전**
- 한국 주식시장은 T+2 결제 방식이므로, 배당을 받으려면 배당기준일 2영업일 전까지 매수해야 함
- 배당기준일은 공공데이터포털 API의 `basDt`(배당기준일자) 필드에서 가져옴
- 영업일 계산 시 공휴일/주말을 고려해야 하므로 한국 공휴일 캘린더 데이터 필요

### 2.3 데이터 수집 전략

```
[일일 배치 작업 흐름]

1. 공공데이터포털 API → 배당기준일, 배당지급일, 배당금액 수집
2. DART OpenAPI → 사업보고서 배당 섹션에서 과거 이력 수집
3. KRX → 시가총액, 종목 기본정보 매핑
4. 배당락일 자동 계산 (배당기준일 - 1영업일)
5. DB 저장 및 캐싱
```

## 3. 주요 기능

### 3.1 배당 캘린더 (메인 페이지)

사용자가 웹앱에 진입하면 가장 먼저 **달력 뷰**가 표시됩니다.

#### 캘린더 모드 선택

- **배당 입금일 캘린더**: 각 날짜에 해당일에 배당금이 입금되는 종목 표시
- **배당락일 캘린더**: 각 날짜에 배당락이 발생하는 종목 표시
- 상단 토글/탭으로 두 모드 간 전환

#### 캘린더 날짜별 표시 규칙

- 각 날짜 셀에는 **최대 3개 종목**만 표시 (시가총액 상위 순)
- 표시 정보: 종목명 + 배당금(주당)
- 3개 초과 시 "+N개 더보기" 표시

#### 날짜 클릭 시 팝업 (상세 보기)

날짜를 클릭하면 모달/바텀시트로 해당 날짜의 전체 배당 이벤트 목록을 표시합니다:

- 종목명, 종목코드
- 주당 배당금
- 배당수익률
- 시가총액
- 배당기준일
- 배당 입금일 / 배당락일
- 전년 대비 배당금 증감 (↑↓ 표시)

#### 월 네비게이션

- 이전/다음 월 이동 버튼
- 연도 선택 드롭다운
- 오늘 날짜 바로가기

### 3.2 종목 검색 및 조회

#### 검색 기능

- 종목명 또는 종목코드로 검색
- 자동완성(Autocomplete) 지원
- 최근 검색 종목 표시

#### 종목 상세 페이지

종목을 선택하면 해당 종목의 배당 관련 상세 정보를 표시합니다:

**기본 정보 영역**
- 종목명, 종목코드, 시장구분(KOSPI/KOSDAQ)
- 현재가, 시가총액
- 배당수익률

**다음 배당 일정**
- 다음 배당기준일
- 다음 배당락일
- 예상 배당 입금일

**배당 히스토리 테이블**

| 연도 | 배당기준일 | 배당락일 | 입금일 | 주당배당금 | 배당수익률 | 전년 대비 |
|------|-----------|---------|--------|-----------|-----------|----------|
| 2025 | 12/31 | 12/30 | 04/25 | 2,994원 | 3.2% | +5.1% ↑ |
| 2024 | 12/29 | 12/28 | 04/20 | 2,850원 | 2.8% | +2.5% ↑ |
| 2023 | 12/29 | 12/28 | 04/18 | 2,780원 | 2.5% | -3.1% ↓ |
| ... | | | | | | |

**배당금 추이 차트**
- 연도별 주당 배당금 추이를 바 차트 또는 라인 차트로 시각화
- 배당금 증감률을 시각적으로 표시

## 5. 데이터 모델

### 5.1 DB 스키마

```sql
-- 종목 기본정보
CREATE TABLE stocks (
  id            SERIAL PRIMARY KEY,
  stock_code    VARCHAR(10) UNIQUE NOT NULL,  -- 종목코드 (예: 005930)
  stock_name    VARCHAR(100) NOT NULL,        -- 종목명 (예: 삼성전자)
  market        VARCHAR(10) NOT NULL,         -- 시장구분 (KOSPI / KOSDAQ)
  market_cap    BIGINT,                       -- 시가총액 (원)
  sector        VARCHAR(50),                  -- 업종
  updated_at    TIMESTAMP DEFAULT NOW()
);

-- 배당 이벤트
CREATE TABLE dividend_events (
  id                SERIAL PRIMARY KEY,
  stock_id          INTEGER REFERENCES stocks(id),
  fiscal_year       INTEGER NOT NULL,           -- 사업연도
  dividend_type     VARCHAR(20),                -- 배당종류 (결산/중간/분기)
  record_date       DATE,                       -- 배당기준일
  ex_dividend_date  DATE,                       -- 배당락일 (계산값)
  payment_date      DATE,                       -- 배당지급일 (입금일)
  dividend_per_share DECIMAL(15,2),             -- 주당 배당금
  dividend_rate     DECIMAL(8,4),               -- 배당률 (%)
  dividend_yield    DECIMAL(8,4),               -- 배당수익률 (%)
  stock_type        VARCHAR(10) DEFAULT '보통주', -- 주식종류
  yoy_change        DECIMAL(8,2),               -- 전년 대비 증감률 (%)
  created_at        TIMESTAMP DEFAULT NOW(),
  updated_at        TIMESTAMP DEFAULT NOW(),
  UNIQUE(stock_id, fiscal_year, dividend_type, stock_type)
);

-- 인덱스
CREATE INDEX idx_dividend_payment_date ON dividend_events(payment_date);
CREATE INDEX idx_dividend_ex_date ON dividend_events(ex_dividend_date);
CREATE INDEX idx_dividend_stock ON dividend_events(stock_id);
CREATE INDEX idx_stocks_market_cap ON stocks(market_cap DESC);
```

## 6. API 설계

### 6.1 내부 API 엔드포인트

```
GET  /api/calendar
     ?year=2026
     &month=4
     &type=payment|ex_dividend
     → 월별 캘린더 데이터 (날짜별 상위 3개 종목)

GET  /api/calendar/:date
     ?type=payment|ex_dividend
     → 특정 날짜의 전체 배당 이벤트 목록

GET  /api/stocks/search
     ?q=삼성
     → 종목 검색 (자동완성)

GET  /api/stocks/:code
     → 종목 상세 정보 + 배당 히스토리

GET  /api/stocks/:code/dividends
     ?from=2020&to=2026
     → 종목별 배당 이력 (차트용 데이터)
```

## 8. 고려사항

### 8.1 데이터 관련

- **배당 입금일의 정확성**: 공공데이터포털 API의 `현금배당지급일자` 필드를 사용하나, 일부 기업은 입금일이 늦게 확정되거나 변경될 수 있음. 주기적으로 데이터 갱신 필요
- **배당락일 계산**: 한국 공휴일(대체공휴일 포함)을 정확히 반영해야 함. `date-holidays` 등의 라이브러리 활용 권장
- **중간배당/분기배당**: 결산배당 외에 중간배당, 분기배당을 실시하는 기업도 있으므로 모든 배당 유형을 수집해야 함
- **API 호출 제한**: 공공데이터포털은 개발계정 기준 일 10,000건. 전체 상장 종목(약 2,500개) 대상 일괄 수집 시 페이지네이션 및 호출 최적화 필요

### 8.2 UX 관련

- 배당 일정이 없는 날짜는 깔끔하게 비워두기
- 배당락일은 투자자에게 "매수 마감일" 의미이므로, 시각적으로 주의를 환기하는 색상 사용
- 모바일에서는 달력 셀이 작으므로 종목명 축약 표시 (예: "삼전", "SK하닉")

### 8.3 법적 고려

- 공공데이터포털 API는 무료 사용 가능하나, 이용약관 준수 필요
- KRX 데이터 스크래핑 시 이용약관 확인 필요
- 투자 조언이 아닌 정보 제공 서비스임을 면책 고지에 명시
